# Protocolo de Reunião da Mentoria

**Versão:** 2.0  
**Data de Criação:** 06/03/2026

---

## Visão Geral

Cada reunião de mentoria segue um protocolo rigoroso que garante:

1. **Gravação automática** via Google Meet
2. **Transcrição automática** (Google Meet nativa)
3. **Ata estruturada** para você e cliente
4. **Extração de dados** da transcrição para controle de andamento
5. **Feedback loop** para melhoria contínua do plano

---

## Fluxo de Reunião

```
Antes da Reunião
    ↓
Reunião (1h via Google Meet)
    ↓
Google Meet gera Transcrição Automática
    ↓
Você extrai dados da transcrição
    ↓
Você preenche Ata (template abaixo)
    ↓
Você consolida em Documento de Andamento
    ↓
Você alimenta Feedback Loop
    ↓
Próxima reunião
```

## Checklist de Extração de Dados da Transcrição

Após cada reunião, use este checklist para extrair dados importantes da transcrição automática. Esses dados alimentam o controle de andamento.

### Checklist Geral

- [ ] **Tema confirmado:** O tema planejado foi coberto?
- [ ] **Objetivos atingidos:** Todos os objetivos foram alcançados?
- [ ] **Tempo:** A reunião ficou dentro de 1 hora?
- [ ] **Engajamento:** Cliente participou ativamente?
- [ ] **Dúvidas:** Quantas dúvidas foram levantadas?
- [ ] **Decisões:** Quantas decisões foram tomadas?
- [ ] **Ações:** Quantas ações foram combinadas?

### Checklist de Conteúdo (Extrair da Transcrição)

**Setor Fiscal:**
- [ ] IRPJ/CSLL discutido?
- [ ] PIS/COFINS discutido?
- [ ] SPED Fiscal discutido?
- [ ] EFD-ICMS/IPI discutido?
- [ ] REINF discutido?
- [ ] DCTF/DCTFWeb discutido?
- [ ] GIA discutido?
- [ ] Reforma Tributária mencionada?

**Setor Contábil:**
- [ ] Contas contábeis discutidas?
- [ ] Escrituração contábil discutida?
- [ ] Demonstrações financeiras discutidas?
- [ ] Conciliação contábil discutida?
- [ ] Fechamento mensal/anual discutido?
- [ ] Balancete/Livro Razão discutido?
- [ ] Imposto de Renda PJ discutido?
- [ ] Reforma Tributária mencionada?

**Comum:**
- [ ] Kanban discutido?
- [ ] Delegação discutida?
- [ ] Avaliação de qualidade discutida?
- [ ] Gamificação mencionada?
- [ ] Prazos confirmados?
- [ ] Responsabilidades claras?

### Checklist de Qualidade (Extrair da Transcrição)

**Clareza:**
- [ ] Cliente entendeu os conceitos?
- [ ] Houve necessidade de repetições?
- [ ] Exemplos foram úteis?

**Engajamento:**
- [ ] Cliente fez perguntas?
- [ ] Cliente sugeriu melhorias?
- [ ] Cliente compartilhou experiências?

**Aplicabilidade:**
- [ ] Conteúdo foi relevante para a empresa?
- [ ] Cliente consegue aplicar imediatamente?
- [ ] Há barreiras de implementação?

## Processo Pós-Reunião

### Passo 1: Baixar Transcrição
1. Acesse Google Meet
2. Clique em "Transcrição"
3. Copie o texto completo
4. Salve em arquivo: `transcricao-semana-X-dia-Y-cliente-NOME.txt`

### Passo 2: Preencher Ata
1. Use o template acima
2. Consolide informações da reunião
3. Extraia dados do checklist
4. Salve como: `ata-semana-X-dia-Y-cliente-NOME.md`

### Passo 3: Consolidar em Documento de Andamento
1. Abra o documento de andamento do cliente
2. Adicione resumo da reunião
3. Atualize métricas e KPIs
4. Salve versão atualizada

### Passo 4: Alimentar Feedback Loop
1. Identifique "Pontos de Melhoria Identificados" na ata
2. Registre em documento separado: `feedback-loop-cliente-NOME.md`
3. Consolide aprendizados para próximo cliente

## Dicas Práticas

1. **Transcrição automática:** Google Meet gera automaticamente. Sempre revise para erros de OCR.
2. **Ata concisa:** Não precisa ser muito longa. 1-2 páginas é o ideal.
3. **Ações claras:** Sempre deixe claro quem faz o quê até quando.
4. **Feedback imediato:** Registre pontos de melhoria enquanto a reunião está fresca.
5. **Consolidação:** No final da semana, consolide todas as atas em um documento único.
