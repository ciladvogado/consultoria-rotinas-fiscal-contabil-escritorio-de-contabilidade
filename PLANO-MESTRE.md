## 1. Visão Geral do Projeto

### 1.1 Objetivo

Criar o planejamento completo e modular de uma empresa de consultoria especializada em **planejamento e implantação de rotinas fiscais e contábeis**, atendendo dois perfis de clientes:

- **Perfil 1 — Empresas particulares** que desejam internalizar o departamento de contabilidade, substituindo a terceirização para escritórios externos.
- **Perfil 2 — Escritórios de contabilidade** que desejam padronizar a prestação de serviços para múltiplos clientes.

### 1.2 Escopo Definido

| Dimensão | Definição |
|----------|-----------|
| Regimes tributários | Simples Nacional, Lucro Presumido e Lucro Real (etiquetáveis por tarefa) |
| Portes de empresa | MEI, ME, EPP, Médio Porte, Grande Porte — etiquetáveis |
| Segmentos econômicos | Todos (indústria, comércio, serviços, agro etc.) — etiquetáveis |
| Níveis tributários | Federal, estadual e municipal |
| Escopo fiscal | Apuração de tributos, escrituração fiscal, obrigações acessórias |
| Escopo contábil | Escrituração contábil, conciliações, fechamentos, balancetes, demonstrações financeiras |
| Fora do escopo | Departamento pessoal/eSocial/folha; rotinas financeiras administrativas (contas a pagar/receber, conciliação bancária) |
| Reforma Tributária | Sinalização dos impactos da LC 214/2025 (IBS/CBS) nas rotinas atuais |
| Nível dos POPs | Conceitual/processual (não operacional em sistema) |
| POPs neste momento | Apenas tópicos e estrutura; documentação individual será posterior |
| Base legal | Não incluída nesta versão; será complementada em versão futura |

### 1.3 Entregáveis ao Cliente Final da Consultoria

A consultoria entrega ao seu cliente final **ambos**:

- **Documento/Manual** — Conjunto de documentos padronizados (POPs, catálogo de obrigações, estrutura de controle).
- **Implantação prática com treinamento** — Acompanhamento in loco ou remoto da implantação das rotinas, com capacitação da equipe.

### 1.4 Sistema de Etiquetas (Tags)

Todas as tarefas, obrigações e rotinas serão etiquetáveis nas seguintes dimensões, permitindo filtragem e agrupamento:

| Dimensão da Etiqueta | Exemplos de Valores |
|----------------------|---------------------|
| Regime tributário | Simples Nacional, Lucro Presumido, Lucro Real |
| Porte da empresa | MEI, ME, EPP, Médio Porte, Grande Porte |
| Segmento econômico | Indústria, Comércio, Serviços, Agronegócio, Construção Civil etc. |
| Perfil do cliente | Empresa (internalização) ou Escritório (padronização) |
| Município / UF | Conforme localização do cliente |
| Área | Fiscal, Contábil |
| Periodicidade | Diária, Semanal, Mensal, Trimestral, Anual, Eventual |
| Nível tributário | Federal, Estadual, Municipal |

### Módulo 1 — Visão Geral do Negócio de Consultoria

**Objetivo:** Definir a proposta de valor, os perfis de clientes e o modelo de atendimento da consultoria.

**Conteúdo previsto:**

1.1. Proposta de valor da consultoria
- O que a empresa faz (planejamento + implantação de rotinas fiscais/contábeis)

### Módulo 4 — Estrutura do Kanban

**Objetivo:** Definir a estrutura do quadro Kanban para controle de tarefas, com filtros, dependências e competência mês/ano.

**Conteúdo previsto:**

4.1. Colunas do Kanban (estágios)

| Coluna | Descrição | Regra |
|--------|-----------|-------|
| Backlog | Tarefas previstas para a competência, ainda não iniciadas | Geradas com base no catálogo e na competência selecionada |
| A Fazer | Tarefas liberadas para execução (pré-requisitos atendidos) | Só entra aqui se as dependências estiverem concluídas |
| Em Execução | Tarefa em andamento pelo responsável | Limite de WIP sugerido por colaborador |
| Em Revisão | Tarefa concluída pelo executor, aguardando revisão de qualidade | Revisão conforme framework do Módulo 6 |
| Concluído | Tarefa revisada e aprovada | Registra data de conclusão e status de qualidade |
| Bloqueado | Tarefa impedida por dependência externa ou problema | Exige justificativa e plano de ação |

4.2. Filtros do Kanban

| Filtro | Função |
|--------|--------|
| Competência (mês/ano) | Visualizar tarefas de um período específico |
| Regime tributário | Filtrar por Simples, Presumido, Real |
| Área | Fiscal ou Contábil |
| Responsável | Filtrar por colaborador |
| Periodicidade | Diária, mensal, trimestral, anual |
| Status de prazo | No prazo, próximo do vencimento, atrasado |
| Cliente | (Para escritórios) filtrar por cliente atendido |
| UF / Município | Filtrar por localização |
| Segmento | Filtrar por segmento econômico |

4.3. Representação de dependências entre tarefas

- **Bloqueio visual:** Tarefas com dependências não concluídas ficam no "Backlog" com indicador visual de bloqueio
- **Cadeia de dependências:** Cada cartão exibe a lista de tarefas predecessoras. Ao concluir uma predecessora, a tarefa dependente é movida para "A Fazer"
- **Tarefas paralelas:** Tarefas sem dependência entre si podem estar simultaneamente em "Em Execução" por responsáveis diferentes
- **Subtarefas:** Obrigações complexas podem ser decompostas em subtarefas dentro do mesmo cartão, com checklist interno

4.4. Estrutura do cartão (card) no Kanban

| Campo do Cartão | Descrição |
|-----------------|-----------|
| Título | Nome da obrigação/rotina |
| Código | Referência ao catálogo (Módulo 2) |
| Competência | Mês/ano de referência |
| Prazo legal | Data-limite |
| Prazo interno | Data-limite interna (margem de segurança) |
| Responsável | Colaborador designado |
| Revisor | Colaborador que fará a revisão de qualidade |
| Etiquetas | Todas as tags aplicáveis |
| Dependências | Lista de tarefas predecessoras |
| Subtarefas | Checklist interno (quando aplicável) |
| Anexos | Documentos, recibos, protocolos |
| Observações | Campo livre para anotações |
| Status de qualidade | Resultado da avaliação (Módulo 6) |
| Pontos de gamificação | Pontuação atribuída (Módulo 9) |
| Histórico | Log de movimentações entre colunas com data/hora |

4.5. Sugestões de ferramentas para implementação

| Ferramenta | Vantagens | Considerações |
|------------|-----------|---------------|
| Notion | Kanban nativo, filtros, banco de dados, templates, automações | Curva de aprendizado moderada |
| Trello | Kanban intuitivo, Power-Ups para dependências e automações | Limitações na versão gratuita |
| ClickUp | Kanban + Gantt + dependências nativas + automações | Mais complexo, mas muito completo |
| Monday.com | Visual, automações, dependências, múltiplas visões | Custo mais elevado |
| Asana | Kanban + timeline + dependências + regras de automação | Boa relação custo-benefício |
| Planilha (Google Sheets/Excel) | Baixo custo, flexível, familiar | Sem automações nativas, mais manual |

### Módulo 5 — Modelo de Delegação de Tarefas

**Objetivo:** Definir papéis, responsabilidades e a matriz de delegação para as rotinas fiscais/contábeis.

**Conteúdo previsto:**

5.1. Papéis sugeridos

| Papel | Descrição |
|-------|-----------|
| Gestor Fiscal/Contábil | Responsável geral pelo departamento; define prioridades, acompanha prazos, aprova entregas críticas |
| Analista Fiscal | Executa apurações de tributos, escrituração fiscal, transmissão de obrigações acessórias fiscais |
| Analista Contábil | Executa escrituração contábil, conciliações, fechamentos, elaboração de demonstrações |
| Assistente Fiscal/Contábil | Apoia na classificação, lançamentos, conferências, organização de documentos |
| Revisor / Auditor Interno | Revisa a qualidade das entregas antes da transmissão/finalização |

5.2. Matriz RACI

Para cada obrigação/rotina do catálogo (Módulo 2), será definido:

- **R** (Responsible) — Quem executa
- **A** (Accountable) — Quem responde pelo resultado
- **C** (Consulted) — Quem é consultado
- **I** (Informed) — Quem é informado

5.3. Critérios de delegação

- Complexidade da tarefa vs. senioridade do colaborador
- Carga de trabalho atual (balanceamento)
- Especialização por área (fiscal vs. contábil)
- Rodízio para desenvolvimento de competências
- Segregação de funções (quem executa não revisa)

5.4. Modelo de dimensionamento de equipe

Diretrizes para estimar a quantidade de profissionais necessários com base em:
- Regime tributário do cliente
- Volume de notas fiscais / lançamentos
- Quantidade de obrigações aplicáveis
- Quantidade de clientes (para escritórios)

### Módulo 6 — Framework de Avaliação de Prazos e Qualidade

**Objetivo:** Criar um sistema completo de avaliação do cumprimento de prazos e da qualidade da execução das tarefas.

**Conteúdo previsto:**

6.1. Avaliação de cumprimento de prazos

6.1.1. Indicadores de prazo

| Indicador | Fórmula | Meta sugerida |
|-----------|---------|---------------|
| Taxa de pontualidade | (Tarefas entregues no prazo / Total de tarefas) x 100 | >= 98% |
| Tempo médio de antecipação | Média de (prazo legal - data de conclusão) | >= 3 dias úteis |
| Taxa de atraso | (Tarefas atrasadas / Total de tarefas) x 100 | <= 2% |
| Tarefas em risco | Tarefas a menos de 2 dias úteis do prazo e ainda não concluídas | 0 |

6.1.2. Rotina de monitoramento de prazos

- **Diária:** Verificação do painel Kanban — identificar tarefas próximas do vencimento
- **Semanal:** Reunião rápida (15 min) de status — revisar tarefas da semana, identificar bloqueios
- **Mensal:** Relatório de desempenho de prazos da competência encerrada
- **Sistema de alertas:** Notificações em 3 níveis (verde: 5+ dias; amarelo: 2-4 dias; vermelho: 1 dia ou vencido)

6.2. Framework de avaliação de qualidade

6.2.1. Dimensões de qualidade

| Dimensão | O que avalia | Peso sugerido |
|----------|-------------|---------------|
| Conformidade processual | A tarefa seguiu o POP definido? | 20% |
| Exatidão dos dados | Os valores, cálculos e informações estão corretos? | 30% |
| Completude | Todas as informações necessárias foram incluídas? | 20% |
| Conformidade legal | A entrega está em conformidade com a legislação vigente? | 20% |
| Documentação e rastreabilidade | Os registros e evidências foram devidamente arquivados? | 10% |

6.2.2. Checklist de revisão por tipo de tarefa

Para cada categoria de tarefa (apuração de tributo, escrituração, obrigação acessória, fechamento contábil etc.), será criado um checklist de revisão com itens específicos.

6.2.3. Escala de avaliação de qualidade

| Nota | Classificação | Critério |
|------|--------------|----------|
| 5 | Excelente | Sem erros, POP seguido integralmente, documentação completa |
| 4 | Bom | Erros mínimos e não materiais, POP seguido com pequenos desvios |
| 3 | Satisfatório | Erros identificados na revisão e corrigidos antes da entrega final |
| 2 | Insuficiente | Erros materiais que exigiram retrabalho significativo |
| 1 | Crítico | Erros que geraram ou poderiam gerar penalidades, retificações ou prejuízos |

6.2.4. Processo de revisão (Quality Gate)

- **Autoconferência:** O executor verifica usando o checklist antes de mover para "Em Revisão"
- **Revisão por pares:** Um segundo profissional revisa usando o checklist
- **Aprovação do gestor:** Para tarefas críticas, o gestor aprova antes da transmissão
- **Amostragem:** Para tarefas de menor risco, revisão por amostragem (ex.: 30%)

6.2.5. Indicadores de qualidade

| Indicador | Fórmula | Meta sugerida |
|-----------|---------|---------------|
| Taxa de acerto na primeira entrega | (Tarefas aprovadas sem correção / Total revisado) x 100 | >= 90% |
| Taxa de retificação | (Obrigações retificadas / Total transmitido) x 100 | <= 2% |
| Índice de qualidade médio | Média das notas de qualidade (escala 1-5) | >= 4,0 |
| Taxa de retrabalho | (Tarefas devolvidas para correção / Total revisado) x 100 | <= 10% |
| Incidentes de penalidade | Quantidade de multas/penalidades por atraso ou erro | 0 |

6.2.6. Relatórios e ciclo de melhoria contínua

- Relatório mensal de qualidade
- Reunião mensal de qualidade
- Plano de desenvolvimento individual
- Atualização dos POPs quando erros recorrentes indicarem falha no procedimento
- Benchmarking interno

6.3. Sugestões de ferramentas para avaliação de qualidade

| Ferramenta / Método | Aplicação |
|---------------------|-----------|
| Checklist digital (no próprio Kanban) | Revisão ponto a ponto dentro do cartão da tarefa |
| Formulário de avaliação (Google Forms / Typeform) | Registro padronizado da avaliação |
| Dashboard de indicadores (Google Sheets / Power BI / Metabase) | Visualização consolidada dos KPIs |
| Reuniões de retrospectiva (modelo ágil) | Discussão periódica sobre melhorias |
| Matriz de competências | Mapeamento de gaps para direcionar treinamentos |

### Módulo 7 — Sinalização dos Impactos da Reforma Tributária

**Objetivo:** Identificar, em cada obrigação/rotina do catálogo, os impactos previstos pela LC 214/2025 (IBS/CBS).

**Conteúdo previsto:**

7.1. Contexto da Reforma Tributária do Consumo

- Substituição de PIS, COFINS, IPI (federal) e ICMS (estadual), ISS (municipal) pelo IBS + CBS
- Período de transição (2026-2033)
- Impacto na estrutura de obrigações acessórias

7.2. Classificação do impacto por obrigação/rotina

| Sinalização | Significado |
|-------------|------------|
| Sem impacto | Obrigação não será afetada pela Reforma |
| Será extinta | Obrigação será descontinuada após a transição |
| Será substituída | Obrigação será substituída por nova obrigação do IBS/CBS |
| Será alterada | Obrigação continuará existindo, mas com alterações |
| Nova obrigação | Obrigação que não existe hoje e será criada pela Reforma |
| Em definição | Impacto ainda não regulamentado ou incerto |

7.3. Novas rotinas previstas

- Classificação tributária IBS/CBS (CST, cClassTrib, IndOp)
- Split payment (cisão do pagamento)
- Escrituração e apuração do IBS e da CBS
- Novas obrigações acessórias do IBS/CBS
- Crédito financeiro (não cumulatividade plena)

7.4. Cronograma de transição e marcos relevantes

- 2026: Início da CBS e IBS (alíquota-teste)
- 2027-2028: Coexistência dos tributos atuais com IBS/CBS
- 2029-2032: Redução gradual de PIS/COFINS, ICMS e ISS
- 2033: Extinção completa dos tributos substituídos

7.5. Recomendações para os clientes da consultoria

- Manter rotinas atuais até a efetiva substituição
- Iniciar capacitação sobre IBS/CBS desde já
- Monitorar regulamentação complementar
- Planejar adaptação de sistemas

### Módulo 8 — Módulo Específico para Escritórios de Contabilidade

**Objetivo:** Definir processos e controles adicionais necessários para escritórios que atendem múltiplos clientes.

**Conteúdo previsto:**

8.1. Gestão de carteira de clientes

- Cadastro estruturado de clientes (dados cadastrais, regime, segmento, porte, UF/município, obrigações aplicáveis)
- Visão consolidada: quais clientes têm quais obrigações
- Matriz cliente x obrigação x responsável
- Classificação de clientes por complexidade / nível de serviço
- Controle de contratos e escopo contratado por cliente

8.2. Onboarding de novos clientes

- Documentação societária (contrato social, alterações, CNPJ)
- Procurações eletrônicas (e-CAC, prefeitura, SEFAZ)
- Certificado digital (A1/A3 — validade, acesso)
- Credenciais de acesso a sistemas
- Levantamento de obrigações aplicáveis
- Histórico fiscal/contábil (últimas declarações, pendências, parcelamentos)
- Parametrização no sistema do escritório
- Definição do responsável interno
- Reunião de alinhamento com o cliente
- Criação dos cartões no Kanban para o novo cliente

8.3. Distribuição de carga de trabalho entre colaboradores

- Critérios de alocação (especialização, capacidade, complexidade)
- Limite de clientes por colaborador
- Balanceamento de carga por período
- Plano de contingência (cobertura em caso de ausência)

8.4. Controle de prazos consolidado (visão multi-cliente)

- Painel consolidado: todas as obrigações de todos os clientes, por competência
- Filtros por cliente, responsável, status, prazo
- Visão de calendário mensal com todas as entregas
- Alertas de concentração

8.5. Padronização da prestação de serviço

- Uso dos mesmos POPs para todos os clientes
- Template de relatórios para o cliente
- Comunicação padronizada (modelos de e-mail, notificações)
- Controle de versão dos POPs

8.6. Offboarding de clientes

- Checklist de encerramento
- Carta de responsabilidade técnica
- Arquivamento do histórico

### Módulo 9 — Gamificação

**Objetivo:** Desenvolver um sistema de gamificação que torne transparente o engajamento e o alcance de metas, aplicável a todos os públicos (colaboradores do cliente, equipe do escritório e clientes da consultoria).

**Conteúdo previsto:**

9.1. Públicos-alvo da gamificação

| Público | Objetivo da gamificação |
|---------|------------------------|
| Colaboradores da empresa (Perfil 1) | Engajar no cumprimento das rotinas, prazos e qualidade durante e após a implantação |
| Colaboradores do escritório (Perfil 2) | Engajar na padronização e na excelência da prestação de serviço multi-cliente |
| Clientes da consultoria | Engajar no processo de implantação, fornecimento de informações e adoção das novas rotinas |

9.2. Mecânicas de gamificação

9.2.1. Sistema de pontuação

- **Pontos por tarefa concluída no prazo** — Valor base conforme complexidade da tarefa
- **Pontos bônus por antecipação** — Multiplicador para entregas antes do prazo interno
- **Pontos por qualidade** — Multiplicador conforme nota de qualidade (Módulo 6)
- **Pontos por sequência (streak)** — Bônus por meses consecutivos sem atrasos
- **Penalidade por atraso** — Redução de pontos proporcional ao atraso
- **Penalidade por retrabalho** — Redução de pontos quando a tarefa é devolvida para correção

9.2.2. Níveis e progressão

| Nível | Nome sugerido | Critério de pontuação |
|-------|--------------|----------------------|
| 1 | Iniciante | 0 - 499 pontos acumulados |
| 2 | Praticante | 500 - 1.499 pontos |
| 3 | Proficiente | 1.500 - 3.499 pontos |
| 4 | Especialista | 3.500 - 6.999 pontos |
| 5 | Mestre | 7.000+ pontos |

9.2.3. Conquistas (badges/medalhas)

- **Pontualidade Perfeita** — Mês inteiro sem atrasos
- **Qualidade Ouro** — Nota média >= 4,5 no mês
- **Zero Retificações** — Trimestre sem retificações de obrigações
- **Multitarefa** — Concluiu tarefas em ambas as áreas (fiscal e contábil) no mês
- **Mentor** — Auxiliou na revisão de tarefas de colegas (revisão por pares)
- **Maratonista** — 6 meses consecutivos acima da meta de pontualidade
- **Implantação Concluída** — (Para clientes) Todas as rotinas implantadas e operando

9.2.4. Rankings e visibilidade

- **Ranking individual mensal** — Classificação dos colaboradores por pontuação no mês
- **Ranking por equipe/área** — Comparação entre equipes (fiscal vs. contábil)
- **Ranking por cliente** — (Para escritórios) Qual carteira de clientes tem melhor desempenho
- **Painel de transparência** — Dashboard visível a todos com os indicadores de engajamento

9.3. Estrutura de controle e apontamento

9.3.1. Fontes de dados (de onde vêm os pontos)

| Fonte | Dados capturados | Integração |
|-------|-----------------|------------|
| Kanban (Módulo 4) | Conclusão de tarefas, datas, responsáveis | Automática (movimentação de cartões) |
| Framework de qualidade (Módulo 6) | Notas de qualidade, checklist de revisão | Registro pelo revisor |
| Controle de prazos | Prazo legal, prazo interno, data de conclusão | Calculado automaticamente |
| Registro de retificações | Obrigações retificadas | Apontamento manual ou via sistema |

9.3.2. Estrutura do registro de apontamento

| Campo | Descrição |
|-------|-----------|
| Data | Data do apontamento |
| Competência | Mês/ano de referência |
| Colaborador | Quem executou |
| Tarefa (código) | Referência ao catálogo (Módulo 2) |
| Pontos base | Pontuação base da tarefa |
| Multiplicador prazo | Fator aplicado conforme antecipação/atraso |
| Multiplicador qualidade | Fator aplicado conforme nota de qualidade |
| Pontos finais | Pontos base x multiplicadores |
| Conquistas desbloqueadas | Badges obtidas neste apontamento |
| Observações | Justificativas ou comentários |

9.3.3. Tabela de pontuação sugerida

| Tipo de tarefa | Pontos base | Exemplo |
|----------------|------------|---------|
| Rotina diária | 5 | Classificação contábil |
| Obrigação mensal simples | 20 | PGDAS-D, DAS-MEI |
| Obrigação mensal complexa | 40 | EFD-ICMS/IPI, EFD-Contribuições |
| Fechamento mensal | 50 | Fechamento contábil |
| Obrigação trimestral | 60 | Apuração IRPJ trimestral |
| Obrigação anual simples | 80 | DEFIS, DASN-SIMEI |
| Obrigação anual complexa | 150 | ECF, ECD |
| Fechamento anual | 200 | Demonstrações financeiras completas |

9.3.4. Multiplicadores

| Condição | Multiplicador |
|----------|--------------|
| Entrega com 5+ dias de antecipação | x 1,5 |
| Entrega com 3-4 dias de antecipação | x 1,3 |
| Entrega com 1-2 dias de antecipação | x 1,1 |
| Entrega no prazo (sem antecipação) | x 1,0 |
| Entrega com 1 dia de atraso | x 0,7 |
| Entrega com 2+ dias de atraso | x 0,5 |
| Qualidade nota 5 | x 1,3 |
| Qualidade nota 4 | x 1,1 |
| Qualidade nota 3 | x 1,0 |
| Qualidade nota 2 | x 0,8 |
| Qualidade nota 1 | x 0,5 |

9.4. Gamificação para clientes da consultoria

| Mecânica | Aplicação |
|----------|-----------|
| Checklist de implantação | Cada etapa concluída pelo cliente gera pontos (ex.: entregou documentação, configurou acessos) |
| Prazo de resposta | Pontos bônus quando o cliente responde solicitações da consultoria dentro do prazo |
| Adoção das rotinas | Pontos quando o cliente demonstra que a equipe está executando conforme o POP |
| Nível de maturidade | Progressão do cliente de "Em implantação" para "Operando" para "Excelência" |

9.5. Relatórios de gamificação

- **Painel individual** — Pontuação, nível, conquistas, histórico, posição no ranking
- **Painel da equipe/departamento** — Pontuação agregada, distribuição de níveis, conquistas coletivas
- **Relatório mensal de engajamento** — Evolução dos indicadores, comparativo com meses anteriores
- **Índice de engajamento** — Métrica consolidada que demonstra transparentemente se o sucesso foi alcançado

9.6. Índice de engajamento (transparência do sucesso)

| Faixa do índice | Classificação | Interpretação |
|-----------------|--------------|---------------|
| 90-100% | Engajamento pleno | Sucesso alcançado — rotinas operando com excelência |
| 75-89% | Engajamento alto | Bom desempenho, com pontos de melhoria identificáveis |
| 60-74% | Engajamento moderado | Atenção necessária — risco de queda de qualidade |
| 40-59% | Engajamento baixo | Intervenção necessária — plano de ação urgente |
| 0-39% | Engajamento crítico | Falha na implantação/operação — revisão completa do processo |

O índice é calculado combinando: taxa de pontualidade, índice de qualidade médio, taxa de participação (tarefas executadas vs. previstas) e evolução de nível na gamificação.

9.7. Sugestões de ferramentas para gamificação

| Ferramenta | Aplicação |
|------------|-----------|
| Planilha automatizada (Google Sheets) | Controle de pontos, rankings, dashboards — baixo custo |
| Notion com fórmulas e banco de dados | Integrado ao Kanban, visual, colaborativo |
| Aplicativo dedicado (Habitica, Spinify, Mambo.io) | Gamificação nativa, mas custo adicional |
| Dashboard customizado (Power BI / Metabase) | Visualização avançada dos indicadores |
| Desenvolvimento próprio (webapp) | Máxima personalização, maior investimento |
